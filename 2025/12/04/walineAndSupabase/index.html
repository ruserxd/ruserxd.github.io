# 製作動機

原先是看 8樓部落格學習製作留言板，但發現 Vercel 在建立 storage 的選項上已沒有 postgress，原因，
因此決定使用 Supabase 建立

# Vercel 服務端部署

主要按照Waline文檔的步驟做
按下 deploy -> 登入 github 帳號


創立一個 waline repository 在自己的帳號下
我取名為 waline-test，按下 create，等待部署


部署完後會如上圖，按下 Continue to DashBoard


此外，在自己的 Github 上也會看到一個剛部署好的 private repository

# 創立 Storage

接著要創立 PostgreSQL 的資料庫，用來存放留言的相關資訊


按下 Storage


按下 Create Database


選擇 Supabase 並一直按下 Continue


輸入 database 的名稱，按下 Create

# Supabase 設定


上述步驟完成後，會在自己的 Supabase 看到 supabase-vercel，點進去


左側欄選擇 SQL Editor


把PGSQL複製上並按下 Run
創立好後再 Table Editor 上會看到多了三張 Table


接著在上方按下 Connect


Method 調成 Session pooler

連接資訊要記住(host, port, database, user, pool_mode)

# Vercel 環境變數設定

接著要確定 Vercel 上的環境變數有符合Waline環境變數規定

Settings -> Enviroment Variables


確保有 PG_HOST, PG_DB, PG_USER, PG_PASSWORD, PG_SSL, PG_PORT
PG_SSL(true)
PG_PORT(5432)

# 最後


回到 OverView 並按下 Visit

測試是否可以正常留言，就完成了
